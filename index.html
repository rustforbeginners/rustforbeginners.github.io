Mac & Linux Installation

Depending on your computer's architecture, there are two different ways to install Rust. First, we will cover the installation on Mac.

1. Go to www.rust-lang.org and run the command (in your terminal) under the header: "Windows Subsystem for Linux".

2. Just continue with the "default installation".

Windows Installation

Before starting the Rust installation, you must first install the C++ Build Tools from Microsoft. These will allow you to run your rust program, because Rust is written in C / C++. And since C++ is a super set of C, all C programs will work in a C++ compiler. Follow these steps to install:

1. Go to https://visualstudio.microsoft.com/downloads/ and download the Visual Studio Community Version.

2. Click continue, then you will be met with a box menu of development kits that you can install. Click the C++ desktop development kit option. Then install.

3. Go to www.rust-lang.org and download the correct installer for your computer's architecture. If you are on a 32-bit computer, then download the 32-bit installer, and vise versa. Continue with the "default installation".

Creating Rust Project

Before creating our first program in Rust, we must create a folder with the essential files to compile your Rust program. Here are the steps:

1. Create an empty folder (ex. "rust_projects") and remember it's directory.

2. In your terminal or command prompt, run the cd command: "cd (file_dir)".

3. While in that directory, run the "cargo new (your_project_name)".

4. Open the project folder in your code editor of choice. A good option is Visual Studio Code. (which is what I will be using).

Layout of Rust Project

In your Rust project, you will notice 3 main (root) files: "src", ".gitignore", "Cargo.toml". The src folder contains your main.rs file. This is where all of your Rust code will go. The .gitignore file tells Git what fiels to ignore when creating a repository on GitHub. The Cargo.toml file lists (and you can add to) your project's dependencies (imports). You can use https://crates.io which will tell you the latest version of the crate.

First Program

Usually, your main.rs file comes with some default code in it. This code will output "Hello, World!" to the standard output (usually terminal or command prompt). Outside of this statement, there is fn main() {}. This tells the compiler main() is the main function, and is going to be run. Of course, you can have other functions, but those will just contribute to the main function.

Going to your terminal, you can cd into the src folder, then run the "rustc main.rs" command to compile your main.rs file. However, this will only compile it, and not run it. To run it, go into your code editor, and there should be two new files: "main.exe" and "main.pdb". For right now, only the main.exe file matters. If you are on windows, you must run this in command prompt. Go to your terminal (or cmd) and run main.exe by saying "main.exe" then pressing enter.

Unlike other languages, Rust differentiates default Rust functoins and user-created functions with a "!" at the end of the name. This is called a macro. (println is a reserved function, called a "macro").

printf Equivalent

If you know anything about the C language, you know that you can do a printf("%d", 3). This is basically replacing the (decimal) placeholder (%d) with the value 3. In Rust, there is a printf equivalent. This is the basic "println! / print!" macro.

fn main() {
  let num=3;
  println!("num: {}", num);
}

Run your Rust Program

To run your Rust program, cd into your Rust project's directory. Then, in the standard output window (cmd or terminal), runt the cargo build command to compile your Rust program. After this has finished (compilation should only take about half a second max), run the cargo run command to run your program. At the bottom of the warning messages, you should see two words: "Hello, World!". If you see these words, you have successfully created your first Rust program.

Error Handling

If (actually, when) an error shows up in your code, do not worry. The best thing to do when there is an error is to stay calm and READ THE ERROR MESSAGES. The entire reason that error messages exist is to help you fix your code. Also, unless you are exploiting or abusing Rust, the Rust programming language cannot harm your computer, so know that all errors do not mean that you computer is at risk, they just mean that the program wasn't able to run.

The Binary Number System

Before understanding data types in Rust, you must have a very basic understanding of the binary number system. Binary (bi = 2) uses 1 and 0 instead of 0-9. You may be asking: "Well, how does it know the value of a number?". Well, each digit, going from right to left, holds more and more value, in a genius way. Each digit has either a 1 or 0, which represents if that value is contributing to the value of the final number. Let me explain in an example. If you do not kind-of understand the binary number system after reading the following, please search for a binary tutorial.

Binary (bits=# of digits):
8 bit binary: 00000000

Each digit, going from right to left has a value of:

2 ^ digits_from_end

Example: 0101
Going from right to left, the digits have values of:
0 (8) 1 (4) 0 (2) 1 (1)

Since the "4" and "1" bits are active (have 1s), these
are added to get the final product; which would be 5.
So, 0101 in decimal (our regular number system)
is 5.

Data Types

Warning: without reading "the binary number system", you will not understand the following.

Each variable (in any programming language) is stored in memeory (RAM). Each of these variables have different types, and therefor takes up different amounts of space in memory. For example, an i32 (integer with 32 bytes of memory allocated) is smaller than a i64. (byte=8 bits). Here is a small list of data types in Rust:

  • i8, i16, i32, i64, i128 (int)
  • String
  • float
  • double
